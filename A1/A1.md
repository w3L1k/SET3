В файле `A1.cpp` назодится исзодный код которые решил задачу A1i в codeforces с посылкой под номером `348993695` 

В файле `main.cpp` находится изменный код который уже решает поставленную задачу то есть подсчет точек в двух квадратных областях соглсно графику с коллисетвом точек от 100 до 100000 с шагом 500.

Маленький вадрат будет ограничен по X [0,8;2,1] по Y[0,8;2,2]

Большой квадрат будет ограничен по X [0;3,5] по Y[0;3,5]

Файл `main.cpp` вывод такие данные как
- N колличество точек для которых производился расчет
- Area_Narrow площаль узкой области
- Error_Narrow относительная погрешность для узкой области
- Area_Wide площадь широкой области
- Error_Wide относитльная погрешность широкой области

Файл `main.cpp` сохраняет данные в файле `results.csv` программа `graph.py` использует данные и строит графики соотношения площадей и относительных ошибок, которые находятся в файлах `Figure_1.png` и  `Figure_2.png`


# Анализ графиков
Сходимость оценки площади к точному значению

На первом графике видно, что:

• Узкая область (синяя линия)

Дает заметно меньший разброс значений.

Уже начиная примерно с N ≈ 2000–5000 оценки стабильно колеблются вокруг точной площади S ≈ 0.944517.

Флуктуации остаются в пределах малой величины, явных больших выбросов нет.

• Широкая область (оранжевая линия)

Имеет значительный размах значений, особенно при малых N — встречаются выбросы > 1.2 и < 0.9.

Даже при N до 100 000 колебания заметно крупнее, чем у узкой области.

Сходимость есть, но она медленнее и менее стабильна.

Вывод:
Использование узкой области существенно ускоряет сходимость Монте-Карло и уменьшает разброс результатов.

Поведение относительной погрешности δ

На втором графике:

• Узкая область (синий график)

Относительная погрешность стабильно ниже, чем у широкой области.

Уже с N ~ 10³–10⁴ погрешность опускается до уровня 10⁻² – 10⁻³, что соответствует высокой точности.

При больших N наблюдается хаотическое, но низкоамплитудное колебание вокруг уровня 10⁻³.

• Широкая область (оранжевая линия)

На малых N даёт очень большую погрешность — вплоть до 10⁻¹.

При росте N погрешность уменьшается, но остаётся более нестабильной и в среднем выше, чем у узкой области.

Даже при N = 100k кучность хуже: многие точки лежат в диапазоне 10⁻² – 10⁻¹, что на порядок хуже узкой области.

# Вывод:
Узкая область обеспечивает существенно меньшую относительную погрешность при одинаковом числе точек и демонстрирует более регулярное уменьшение ошибки.

 Сравнение эффективности двух подходов

По графикам отчётливо видно:

Для достижения приемлемой точности (δ < 1%)
– узкая область требует ~5–10 тыс. точек,
– широкая область требует в несколько раз больше точек (порядка 40–80 тыс.).

Разброс результатов у широкой области всегда ощутимо выше — следствие того, что большая часть точек попадает «впустую» вне целевой фигуры.


Узкая прямоугольная область делает метод Монте-Карло намного эффективнее, снижая дисперсию и погрешность при одинаковом N.



По результатам эксперимента можно заключить, что:

Алгоритм Монте-Карло корректно сходится к точной площади пересечения трёх окружностей.

Скорость и стабильность сходимости сильно зависят от выбора области выборки.

Узкая область значительно улучшает точность и уменьшает флуктуации.

Для получения точности порядка 0.01 достаточно N ≈ 20–30 тыс. при узкой области,
тогда как широкая область требует существенно большего числа точек.

Результаты испытаний полностью подтверждают теоретическое ожидание: дисперсия пропорциональна площади области выборки.
