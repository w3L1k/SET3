Вывод по результатам эмпирического анализа MERGE SORT и гибридного MERGE+INSERTION SORT

В рамках исследования были проведены измерения времени работы двух алгоритмов сортировки:

стандартной рекурсивной реализации MERGE SORT;

гибридной реализации MERGE+INSERTION SORT, в которой при размере подмассива, не превышающем пороговое значение threshold, используется Insertion Sort.

файл `A2.cpp` был напрвлен на платформу codeforces посылкой №348994984

файл `Aa2.cpp` был модифицирован в него были добавленны генерация случайных массивов, замер времени и запись результатов в `results.csv` 

Анализ полученных данных был проведен с помощью `graph.py` но при первоначальном анализе было несколько выбросов которые не позволяли провести качественный анализ (графики до отчистки данных находятся в соответсвующей папке), поэтому было проведенна очистка данных и все актуальные графиуи находятся в соотвствующей папке


Эксперименты выполнены для трёх типов данных:

 - случайные массивы (random),

 - почти отсортированные массивы (almost sorted),

 - массивы, отсортированные в обратном порядке (reversed).

Размеры входных данных варьировались от 500 до 100 000 элементов. Для гибридного алгоритма рассматривались пороги переключения: 5, 10, 20, 30 и 50.

1. Сравнение MERGE SORT и MERGE+INSERTION SORT

По всем графикам видно, что гибридный алгоритм в среднем работает быстрее классического Merge Sort.
Особенно это проявляется на больших массивах.

Основная причина заключается в том, что при рекурсивном разбиении Merge Sort формирует большое количество небольших подмассивов, на которых Insertion Sort работает эффективнее.
Гибридный алгоритм сокращает накладные расходы рекурсии и быстрее обрабатывает эти небольшие участки.

2. Оптимальное значение порога threshold

Графики зависимости времени от значения threshold показывают, что:

при threshold = 5 ускорение относительно чистого Merge Sort есть, но оно незначительно;

значения threshold = 10 и threshold = 20 дают наилучшие результаты;

дальнейшее увеличение порога (30 и 50) приводит к росту времени, так как Insertion Sort начинает применяться слишком часто.

Таким образом, оптимальный диапазон порогов для гибридного алгоритма — от 10 до 20.

3. Влияние типа входных данных
Случайные массивы

Гибрид работает быстрее классической версии примерно на 15–25%.
Минимальное время достигается при threshold = 10–20.

Почти отсортированные массивы

Этот тип данных находится ближе всего к лучшему случаю для Insertion Sort, поэтому гибрид показывает наиболее заметное ускорение — до 25–30%.
Оптимальное значение порога остаётся тем же.

Массивы, отсортированные в обратном порядке

Несмотря на то, что Insertion Sort работает на них хуже, гибрид всё равно оказывается быстрее обычного Merge Sort при пороге 10–20.
При threshold ≥ 30 время начинает расти.

4. Поведение времени при увеличении размера массива

Графики зависимости времени от размера массива показывают:

рост времени соответствует асимптотике O(n log n), что согласуется с теорией;

кривые для гибридного алгоритма лежат ниже кривой для чистого Merge Sort;

разница в скорости увеличивается с ростом размера входных данных;

графики гладкие и устойчивые, что подтверждает корректность проведённых измерений.

5. Общий итог

Гибридный алгоритм MERGE+INSERTION SORT стабильно превосходит по скорости классический MERGE SORT.

Оптимальное значение порога переключения находится в диапазоне 10–20.

Максимальный прирост производительности наблюдается для почти отсортированных данных, минимальный — для обратного порядка.

Слишком большие пороги (threshold ≥ 30) приводят к ухудшению времени работы за счёт чрезмерного использования Insertion Sort.

Алгоритмы демонстрируют поведение, соответствующее заявленной сложности O(n log n).
